How PolyPulse Works

PolyPulse watches public Polymarket trade data and turns it into two research views: a live-style trade feed and a win chance tracker. It is built for scanning market flow, not for automatic trading or guaranteed profit.

Whale Trade Tracker

The Feed page lists individual Polymarket trades from the stored data file. It shows trade size, market question, buy or sell side, time, wallet profile link, and a size tier.

Use the period filter and whale filters to focus on recent trades or larger trade sizes. The summary cards update based on the active filters.

Open Feed

Win Chance Tracker

The Signals page groups trades by market outcome and ranks possible ideas using recent buy/sell flow, trade size, market price, and estimated edge.

A high rank means the recent stored flow looks interesting by the simple model. It does not mean the bet will win.

Data And Refresh

Source

The scanner reads Polymarket's public Data API `/trades` endpoint, normalizes the latest trades, and saves them into `data/whales.json`.

Refresh

GitHub Actions updates the stored JSON on a schedule. The website then fetches that file in the browser, so new results appear after the scanner has run and GitHub Pages has served the new file.

Filters

Period

Choose last day, last 7 days, last month, last year, or all stored trades. These filters only search trades currently stored in the JSON archive.

Trade Size

Turn tiers on or off to focus on large positions or include smaller activity. The default minimum stored signal is currently $100.

Win Chance Calculation

Market price is treated as the current implied chance for an outcome.

Flow bias compares recent buy volume with sell volume.

Estimated chance is a simple adjustment from price, flow, and volume. It is only a research shortcut.

flow_bias = (buy_volume - sell_volume) / total_volume
estimated_chance = price + flow_bias * 12% + volume_bonus
edge = estimated_chance - price
score = edge + volume_strength + max_trade_strength

Important Limits

  • Large trades are not automatically smart trades.
  • The signal score does not include news, settlement rules, liquidity depth, hedging, or wallet history.
  • One trader can split activity across many trades or wallets.
  • Results depend on Polymarket's public API, GitHub Actions, and the stored archive size.
