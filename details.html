About this dashboard

What PolyPulse shows

PolyPulse monitors public Polymarket trade history and turns recent trades into a filterable feed. It is meant to help you spot larger trades, repeated activity, and markets with notable recent flow.

The dashboard is informational only. It does not know whether a trade is smart, profitable, hedged, or part of a larger strategy.

Data Source

The scanner uses Polymarket's public Data API `/trades` endpoint. A scheduled GitHub Actions job writes the latest normalized trades into `data/whales.json`, which the static page reads in the browser.

Refresh Model

The backend scanner is scheduled to run every 5 minutes. The browser checks the JSON feed every 30 seconds, so a new trade appears after the scanner has fetched and committed it.

Filters

Period

Choose last day, last 7 days, last month, last year, or all stored trades. The filter only works on trades currently stored in the JSON archive.

Tier

Enable or disable trade-size tiers to focus on larger positions or include smaller pulse-level activity.

Tier Definitions

Tier Trade Size Meaning
Blue WhaleOver $50,000Very large position.
Whale$10,000-$50,000Large high-conviction-size trade.
Shark$5,000-$10,000Significant active trader size.
Dolphin$1,000-$5,000Larger than typical retail size.
Pulse$100-$1,000Smaller signal that keeps the feed active during quieter periods.

Limitations

  • The feed depends on Polymarket's public API and GitHub Actions availability.
  • Large trades are not automatically good trades.
  • One wallet may split activity across multiple trades or addresses.
  • The archive grows over time and is capped by the configured maximum stored trades.
